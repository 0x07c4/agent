# PROFILE

## User Role
- 个人开发者，主要在 Linux 环境下工作。
- 正在构建 `Solo`，一个面向个人开发的人机协作式 AI 工作台。

## Technical Level
- 熟悉前端、桌面端、命令行与代码仓库工作流。
- 能快速判断产品和工程实现是否偏离目标，不需要被过度解释。

## Communication Preferences
- 默认使用中文。
- 直接、简洁、少废话。
- 不要空泛地“讲方案”，优先给能落地的判断和改动。
- 如果方向错了，要直接指出，不要替错误实现找理由。

## Working Preferences
- 明确偏好 human-in-the-loop，不喜欢自治 agent 接管决策。
- 产品路线固定为“对话 + 工作区协作”，核心协作方式是“建议 + 预览 + 用户确认”。
- 视觉上偏好开发工具气质，喜欢吸收 ChatGPT、Copilot、Zed、Neovim、AstroNvim、LazyVim 的优点。
- 重视性能，不希望产品逐渐滑向沉重的伪 IDE。
- 能留在前端的逻辑尽量留在前端，Rust 更适合作为系统桥接层。
